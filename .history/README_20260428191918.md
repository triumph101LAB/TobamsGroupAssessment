# Card Validation API

A REST API that validates card numbers using the Luhn algorithm. 
Built with NestJS and TypeScript, scoped to card networks commonly 
used in the Nigerian market.

## Supported Networks
- Visa
- Mastercard
- Verve

## Requirements
- Node.js v18+
- npm

## Installation

git clone <your-repo-url>
cd <project-folder>
npm install

## Running the Server

npm run start:dev

Server runs on http://localhost:3000

## Running Tests

npm run test

## API Reference

### Validate Card

**POST** `/validate-card`

**Request Body**
```json
{
  "cardNumber": "4111111111111111"
}
```

**Valid Card Response — 200**
```json
{
  "cardNumber": "4111111111111111",
  "length": 16,
  "isValid": true,
  "Network": "Visa"
}
```

**Invalid Card Response — 200**
```json
{
  "cardNumber": "4111111111111112",
  "length": 16,
  "isValid": false,
  "Network": "Visa"
}
```

**Bad Input Response — 400**
```json
{
  "message": "No valid digits found",
  "statusCode": 400
}
```

## Design Decisions

**Framework — NestJS**
NestJS provides a structured, modular architecture out of the box. 
For a project requiring clear separation between controller, service, 
and validation layers, it was the more appropriate choice over Express.
Also, I have been using Nestjs for a while now, and I am more comfortable with it.

**Network Scope**

Scoped to Visa, Mastercard, and Verve — the three dominant card 
networks in the Nigerian market. Verve is included specifically 
because it is the most widely used domestic network, issued by 
fintechs like OPay, Moniepoint, and PalmPay. and also the card lenghts are used are 16, 19.
The geneal standard is 16, but some networks used 19

**Sanitization Strategy**
Spaces and dashes are stripped rather than rejected. Card numbers 
are commonly formatted with spaces for readability. Stripping them 
makes the API more practical for real-world frontend usage.

**Luhn Algorithm**
The Luhn algorithm validates the structural integrity of a card 
number — it catches typos and randomly generated numbers. It does 
not confirm the card exists or has funds. It is a checksum, not 
an authentication mechanism.
